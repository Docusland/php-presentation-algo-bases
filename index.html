Algorithmie Kesaco ?

Un algorithme est une méthode générale pour résoudre un type de problèmes. Il est dit correct lorsque, pour chaque instance du problème, il se termine en produisant la bonne sortie, c'est-à-dire qu'il résout le problème posé.

Les fonctions

Une fonction est un bloc d'instructions que l'on peut utiliser n'importe où dans un programme. Une fonction est caractérisée par plusieurs choses:

  • son nom
  • son type de retour
  • ses paramètres

La fonction max(tableau)

									
										function maximum(array $tableau) : ?int {
											$maximum = null;
											for($i=0; $i < count($tableau); $i++) {
												if($tableau[$i] > $maximum){
													$maximum = $tableau[$i];
												}
											}
											return $maximum;
										}

L'algorithme d'euclide

Ah le PGCD.. que de bons souvenirs...Vous vous souvenez de la règle ?

L'algorithme d'euclide

L'algorithme part du constat suivant : le PGCD de deux nombres n'est pas changé si on remplace le plus grand d'entre eux par leur différence.
Autrement dit, pgcd(a, b) = pgcd(b, a - b)

pgcd(333,185) => 333-185 = 148 => pgcd(185, 148)
(148, 37) => (111, 37) => (74, 37) => (37,37)

37x9 = 333
37x5 = 185 

								
									function pgcd(int $a, int $b) : int {
										while ($a != $b) {
											if($a > $b) {
												$a = $a - $b;
											} else {
												$b = $b - $a;
											}
										}
										return $a;
									}

Les fonctions récursives

Une fonction récursive est une fonction qui pour fournir un résultat, s’appelle elle-même un certain nombre de fois.

L'algorithme d'euclide

									
										function pgcd(int $a, int $b) : int {
											if ($a == $b) return $a;
											if ($a > $b) 
												return pgcd($b, $a - $b);
											else 
												return pgcd($a, $b - $a);
										}

Les bonnes pratiques

Favoriser la lisibilité du code

La programmation est un art et vous passerez plus de temps à lire du code qu'à en produire. Alors appliquez-vous. Ce sera plus agréable pour vous ainsi que pour vos collègues.

Prenez le temps de bien nommer les choses

Une variable nommée `toto` ou `valeur1` n'a aucun sens et vous oblige à mettre un commentaire. Si vous la nommez `nom_utilisateur` la variable est bien plus claire et vous ne vous perdrez pas dans votre code.
